# 跨境电商 + AI视频推广 团队协作工作计划

> 项目周期：2026年3月 - 2026年5月（12周冲刺）
> 团队规模：3人
> 核心方向：为跨境卖家提供AI视频本地化 + 智能推广服务

---

## 一、项目定位

利用可灵3.0多语言口型同步 + DeepSeek低成本文案生成，为TikTok Shop / Amazon跨境卖家提供：
- 一键生成多语言产品推广视频（中/英/日/韩/西班牙语）
- AI驱动的产品listing优化
- 视频投放策略自动化建议

目标客户：中国跨境卖家（首批聚焦TikTok Shop卖家）

---

## 二、团队角色分工

### 成员A — 技术开发（Tech Lead）
- 核心职责：产品后端架构、API对接、视频生成pipeline
- 技术栈：Python / Node.js、可灵API、DeepSeek API、FFmpeg
- 关键任务：
  - 视频生成工作流搭建（可灵3.0 API对接）
  - 多语言文案生成模块（DeepSeek API）
  - 用户系统与订单管理后端
  - 部署与运维

### 成员B — 产品前端 & 设计（Product & Frontend）
- 核心职责：产品设计、前端开发、用户体验
- 技术栈：React / Next.js、Figma、TailwindCSS
- 关键任务：
  - 产品原型设计与UI
  - 前端页面开发（上传、预览、下载流程）
  - 落地页与营销素材制作
  - 用户反馈收集与产品迭代

### 成员C — 运营增长 & 商务（Growth & Biz）
- 核心职责：市场调研、客户获取、内容运营、商务拓展
- 关键任务：
  - 目标客户画像与竞品分析
  - TikTok Shop卖家社群渗透（跨境电商群、知识星球等）
  - 种子用户获取与维护（首批50个付费用户）
  - 定价策略与商业模式验证
  - 内容营销（小红书/公众号/TikTok展示案例）

## 四、GitHub 项目管理规范

### 仓库结构建议

```
├── .github/
│   ├── ISSUE_TEMPLATE/       # Issue模板
│   │   ├── bug_report.md
│   │   ├── feature_request.md
│   │   └── task.md
│   └── PULL_REQUEST_TEMPLATE.md
├── docs/                     # 产品文档、API文档、会议纪要
├── src/
│   ├── backend/              # 后端服务
│   ├── frontend/             # 前端项目
│   └── scripts/              # 工具脚本
├── assets/                   # 设计稿、营销素材
└── README.md
```

### 分支策略

- `main` — 稳定版本，仅通过PR合并
- `dev` — 开发主分支，日常合并目标
- `feature/xxx` — 功能分支（如 `feature/video-pipeline`）
- `fix/xxx` — 修复分支
- `release/vX.X` — 发版分支

### Issue & Project Board 管理

使用 GitHub Projects（看板模式），设置以下列：

| 列名 | 说明 |
|------|------|
| 📋 Backlog | 待排期任务 |
| 🎯 Sprint Todo | 本周待做 |
| 🔨 In Progress | 进行中（每人同时最多2个） |
| 👀 Review | 等待Review/测试 |
| ✅ Done | 已完成 |

### Label 体系

| Label | 用途 |
|-------|------|
| `tech` | 技术开发任务 |
| `product` | 产品设计任务 |
| `growth` | 运营增长任务 |
| `P0-urgent` | 紧急/阻塞性 |
| `P1-high` | 高优先级 |
| `P2-normal` | 常规优先级 |
| `bug` | 缺陷 |
| `milestone-1` / `2` / `3` | 对应三个阶段 |

## 五、协作节奏

### 每日
- **异步站会**（GitHub Discussion 或微信群）：每天早上发当日计划，晚上发进度
- 格式：`昨天完成 / 今天计划 / 阻塞问题`

### 每周
- **周一**：Sprint Planning — 从Backlog拉取本周任务到Sprint Todo
- **周五**：周复盘 — 15分钟线上会，同步进度，调整优先级
- 周复盘纪要提交到 `docs/weekly/` 目录

### 每阶段
- Phase结束时做里程碑Review
- 更新README中的项目进度
- 复盘文档归档到 `docs/review/`

## 七、风险预案

| 风险 | 应对策略 |
|------|---------|
| 可灵API调用限制/不稳定 | 备选方案：万兴播爆API、Runway作为降级 |
| 视频生成成本超预期 | 设置用量上限，优先服务付费用户 |
| 种子用户获取困难 | 免费试用策略，先做10个标杆案例 |
| 关税政策进一步变化 | 保持产品灵活性，支持多平台多地区 |
| 团队成员时间冲突 | 异步协作为主，关键节点同步对齐 |
